Задание 1
Возьмите установочный файл maven. Настройте environment variables и
проверьте, что maven настроен корректно. Напишите версию Maven.    

    Установочный файл maven был скачен в архиве zip, и распаковал в директорию d:\java\apache-maven-3.6.3.
    Чтобы начать использовать maven нужно настроить переменные среды. Это можно сделать двумя способами. 
    В первом мы настраиваем один раз через «мой компьютер – свойство – дополнительные параметры – переменные 
    среды» путем добавления переменной M2_HOME c значение d:\java\apache-maven-3.6.3 и в системной переменной 
    PATH добавляем переменную maven %M2_HOME%\bin. Во втором способе нам приходиться указывать переменные среды каждый 
    раз при запуске командной строки. Для этого в командной строке пишем mvn --version. 
    Если получаем сообщение «mvn не является внутренней или внешней командой, исполняемой программой или пакетным файлом» 
    или нас не устраивает версия maven устанавливаем переменные среды вручную (только для этой сессии) следующий способом:
    set M2_HOME=d:\java\apache-maven-3.6.3
    set PATH=%PATH%;%M2_HOME%\bin
    После этого проверяем версию maven:
    mvn --version
    Если появиться сообщение:
    The JAVA_HOME environment variable is not defined correctly
    This environment variable is needed to run this program
    NB: JAVA_HOME should point to a JDK not a JRE
    То прописываем еще и переменные среды java:
    set JAVA_HOME=d:\java\jdk-11
    set PATH=%PATH%;%JAVA_HOME%\bin
    После этого опять проверяем версию maven и если все правильно сделано получаем следующее сообщение: 
    Apache Maven 3.6.3 (cecedd343002696d0abb50b32b541b8a6ba2883f)
    Maven home: d:\java\apache-maven-3.6.3\bin\..
    Java version: 11, vendor: Oracle Corporation, runtime: d:\java\jdk-11
    Default locale: ru_RU, platform encoding: Cp1251
    OS name: "windows 10", version: "10.0", arch: "amd64", family: "windows"
